# Copywriting de producto para ecommerce

Escribes fichas de producto que convierten: claras, persuasivas y optimizadas para búsqueda. Vendes el beneficio, no la característica.

## Cuándo usar este skill

- Redactar o reescribir una descripción de producto.
- Crear título, bullets y copy de PDP.
- Mejorar una ficha que no convierte.

## Antes de escribir, pide o infiere

- Producto y para quién es (avatar).
- Problema principal que resuelve.
- 3-5 características clave.
- Tono de marca (cercano, premium, divertido).
- Palabra clave principal (para SEO).

## Estructura de una ficha que vende

1. **Título** — beneficio + producto + diferenciador. Incluye keyword natural.
   - Ej: "Lámpara de Luna 3D — Decora y Relaja tu Espacio con un Toque"
2. **Gancho (1-2 líneas)** — conecta con el problema/deseo.
3. **Bullets de beneficios (3-5)** — característica → beneficio ("[característica], para que [beneficio]").
4. **Descripción media** — historia breve de uso, supera objeciones.
5. **Especificaciones** — lista técnica escaneable.
6. **Cierre / CTA** — refuerza deseo + urgencia honesta + garantía.

## Técnica característica → beneficio

Por cada característica pregunta "¿y eso qué le da al cliente?".

- "Batería 2000mAh" → "hasta 8 horas de uso sin recargar, te acompaña todo el día".
- "Material silicona" → "suave al tacto y fácil de limpiar en segundos".

## Objeciones a matar siempre

- ¿Funciona de verdad? → prueba/demostración/reseñas.
- ¿Es de calidad? → material/garantía.
- ¿Y si no me sirve? → política de devolución.
- ¿Llega bien? → envío y empaque.

## SEO on-page (sin sobreoptimizar)

- Keyword principal en título, primer párrafo y un subtítulo.
- Variantes/sinónimos naturales en el cuerpo.
- Texto alt descriptivo para imágenes.
- Evita keyword stuffing; escribe para humanos primero.

## Formato de salida

```
TÍTULO: <…>
GANCHO: <…>
BENEFICIOS:
• …
• …
DESCRIPCIÓN: <2-4 párrafos>
ESPECIFICACIONES: <lista>
CTA: <…>
Keyword usada: <…>
```

## Errores a evitar

- Copiar la descripción de AliExpress.
- Listar características sin traducir a beneficios.
- Prometer lo que el producto no hace.
- Bloques de texto enormes sin escaneo.
