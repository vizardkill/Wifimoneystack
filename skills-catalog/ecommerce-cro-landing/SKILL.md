# CRO de Landing Pages y PDP

Subes la conversión de páginas que reciben tráfico pago. Cada punto de CR equivale a bajar el CPA: optimizar la página suele rendir más que tocar la campaña.

## Cuándo usar este skill

- Diseñar/estructurar una landing de un solo producto.
- Subir conversión de una PDP que recibe tráfico de Meta/TikTok.
- Priorizar qué probar (A/B).
- Ordenar el copy y los elementos de la página.

## Anatomía de una landing que convierte (orden)

1. **Above the fold**: titular de beneficio + imagen/video del producto en uso + CTA + 1 señal de confianza. La promesa debe entenderse en 5 segundos.
2. **Problema/agitación**: conecta con el dolor del avatar.
3. **Solución**: presenta el producto como el puente al resultado deseado.
4. **Beneficios visuales**: 3-4 bloques (imagen + beneficio), no características sueltas.
5. **Prueba social**: reseñas con foto, UGC, antes/después, número de clientes.
6. **Demostración**: GIF/video corto de cómo funciona.
7. **Oferta**: precio con ancla, bundle, garantía, envío.
8. **Objeciones/FAQ**: responde las 5 dudas que frenan la compra.
9. **CTA final** + urgencia/escasez honesta.

## Principios de CRO

- **Una página, un objetivo, un CTA repetido** (mismo texto, mismo color).
- **Claridad > creatividad**: si confunde, no convierte.
- **Móvil primero**: 80%+ del tráfico pago es móvil.
- **Velocidad**: cada segundo de carga cuesta conversión.
- **Fricción mínima**: menos campos, menos clics, métodos de pago locales.

## Priorizar pruebas A/B (modelo ICE)

Puntúa cada idea por Impacto, Confianza y Facilidad (1-5). Prueba primero lo de mayor ICE. Elementos de mayor impacto típico: **titular**, **oferta/precio**,
**imagen/video principal**, **CTA**, **prueba social**.

### Reglas de testeo

- Una variable a la vez.
- Tráfico suficiente para significancia (no concluyas con 30 visitas).
- Mide conversión real, no clics.

## Diagnóstico rápido por síntoma

- Mucho tráfico, poca venta → above the fold débil u oferta poco clara.
- Agregan al carrito, no compran → fricción en checkout o falta de confianza.
- Rebote alto → velocidad, congruencia anuncio↔landing, o promesa confusa.

## Formato de salida

```
DIAGNÓSTICO: <síntoma → causa probable>
ESTRUCTURA SUGERIDA: <bloques en orden>

- CTA: …
PRUEBAS PRIORIZADAS (ICE):
1. … (I/C/E)
2. …
KPI: tasa de conversión / CPA
```
